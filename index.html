História dos computadores

Voltar

A história da computação se iniciou muito antes da criação dos primeiros computadores, com a criação de algoritmos e máquinas de cálculo. Durante a segunda guerra mundial Alan Turing criou o primeiro computador para decodificar as mensagens enviadas pelo exército alemão. Os primeiros computadores tinham como objetivo resolver problemas matemáticos especificos e sua programação era muito mais complexa. Eles ocupavam um espaço enormes e eram operados apenas por profissionais especializados. Com o avanço da tecnologia, os computadores foram diminuindo e tendo cada vez mais funções. Sistemas operaionais foram criados e a interatividade com o usuário, através de telas, teclado e mouse, possibilitaram a utilização dos computadores por pessoas leigas, uma vez que a programação foi escondida em sistemas e aplicativos e só mostram ao usuário o resultado final. A criação da internet foi dada a partir da necessidade de transferência de arquivos entre computadores, partindo da comunicação de código morse até chegar em uma transmissão entre dispositivos. Como padrão foram definidos protocolos, linguagens e parametros para garantir a comunicação entre todos os dispositivos da mesma forma,alguns exemplos de padronizações são tcp-ip, http, www, etc.

O que são Clients?

Voltar

Segundo o ponto de vista do HTML clients, também chamados de clientes, são aqueles que consomem informações da internet ou de outro tipo de rede de informação. Para isso são necessários dispositivos como celulares, computadores e tablets. Arquivos estáticos são enviados do servidor para o cliente, depende da velocidade de conexão do cliente e do tamanho dos arquivos. Exemplos de arquivos estáticos: HTML, CSS, Javascript, imagens... Esses arquivos estáticos são armazenados no cache dentro do navegador do cliente para diminuir o trafego de dados com informações pré carregadas, o cache é sobreescrito quando a página é recarregada. Ao desenvolver um site ou aplicação deve-se considerar os diferentes dispositivos que podem acessar os sites, isso é chamado de Design Responsivo ou responsividade. Boas práticas e testes: verificar compatibilidade com diferentes navegadores, podem haver comportamentos diferentes no mesmo dispositivo de acordo com a interpretação do navegador.

Navegadores:

São programas instalados no dispositivo que interpretam os arquivos que vem do servidor web. Também chamados de browsers. Seguem o padrão W3C, base é a mesma para todos. O que significa que as tags mais básicas devem funcionar em todos os navegadores São ferramentas para utilizar a internet.

Aplicações Web

Aplicações web são soluções criadas que possuem a internet como meio de comunicação entre client x server não sendo necessária a sua instalação. Elas são acessadas através do navegador. Apps instalados tem outra abordagem de programação. Existem semelhanças, mas são diferentes na forma de programação.

O que são Servers?

Voltar

Os Servers, ou servidores, são computadores mais robustos do que os de usuários, eles exigem processamento maior e servem a vários dispositivos. Ele pode ser ligado a internet ou pode ser conectado a uma rede local LAN, geralmente empresas, computadoor central é um servidor. Pode se conectar com bancos de dados. Sites são hospedados em servidor. Data centers são conjuntos de servidores.

Alguns tipos de servidores são:

  • Arquivos (central de imagens, videos, etc)
  • Segurança (Firewall)
  • Streaming (principalmente videos)
  • E-mail (Armazena e envia os e-mails)
  • Web

Qual a estrutura básica de um HTML?

Voltar

O arquivo index é o padrão de pasta principal do site. As tags apresentam a seguinte estrutura:

  • < nome_da_tag> : abre
  • < /nome_da_tag>: fecha
As tags principais:
  • html: tudo entre ela é uma página html
  • head: carrega antes de exibir para o usuário
  • title: nome que aparece na aba
  • body: aparece para o usuário
Uma estrutura básica ficaria:
< html>
< head>
    < title> Aqui coloca-se o título < /title>
< /head>
< body>
    Aqui vai todo o conteúdo do site
< /body>
< /html>

Entendendo a diferença entre listas ordenadas e não-ordenadas

Voltar

Listas ordenadas são aquelas em que os elementos são identificados em com marcadores ordenados,utilizados em rankings, modo de preparo de receitas entre outros. Por exemplo:

  1. Elemento 1
  2. Elemento 2
  3. Elemento 3
Já as listas não-ordenadas são identificadas por marcadores iguais, esse tipo de lista é utilizado em tópicos e em listas em que a ordem dos elementos não tem tanta importância. Por exemplo:
  • Item A
  • Item B
  • Item C
